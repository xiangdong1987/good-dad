# good-dad 设计系统 · 圆润奶油风

> 目标：妈妈一眼觉得"这个 App 真懂我"，爸爸一眼就知道"现在我该做什么"。
> 关键词：**温柔、清晰、踏实**。可爱但不幼稚，专业但不冷漠。

---

## 1. 设计原则

| # | 原则 | 落地表现 |
|---|---|---|
| 1 | **一屏看完核心** | 首页固定区不超过 3 张大卡片：今日重点 / 下一步行动 / 妈妈状态 |
| 2 | **大手指友好** | 主要按钮 ≥ 56dp 高，间距宽松，爸爸单手可操作 |
| 3 | **温柔不卖萌** | 圆润但不卡通，使用插画式贴纸而非 emoji 堆砌 |
| 4 | **数据要"说人话"** | "宝宝今天比昨天多睡了 20 分钟，做得好" 优于 "睡眠 +20min" |
| 5 | **AI 像贴心嫂子** | 助手回答带温度，不端医生架子，但关键医学信息一定要保守 |

| Token | HEX | 用途 |
|---|---|---|

## 3. 字体

```
中文：HarmonyOS Sans / 思源黑体 → 圆润 fallback：阿里妈妈方圆体 VF
英文/数字：Inter（数字用 tabular-nums）
```

| 名称 | 字号 / 行高 | 字重 | 用途 |
|---|---|---|---|
| `display`  | 32 / 40 | 600 | 周次大数字、孕周提示 |
| `title.lg` | 22 / 30 | 600 | 页面主标题 |
| `title.md` | 18 / 26 | 600 | 卡片标题 |
| `body.lg`  | 16 / 24 | 400 | 正文（爸爸更容易读） |
| `body.md`  | 14 / 22 | 400 | 次要正文 |
| `label`    | 13 / 18 | 500 | 标签、Chip |
| `caption`  | 12 / 16 | 400 | 时间戳、说明文字 |

> 不使用 < 12sp 的字号；爸爸很多在通勤、夜里看，必须保证可读。

## 5. 核心组件规范

### 5.1 CreamCard（基础卡片）
- 底色 `cream.100`，圆角 `radius.lg`，阴影 `shadow.soft`
- 内边距 `spacing.lg`
- 顶部可选"贴纸标签"（小色块 + 圆头插画图标）

### 5.2 PrimaryButton
- 背景 `peach.500`，文字白色 `body.lg / 600`
- 高度 56，圆角 `radius.pill`
- 按下：背景 `peach.700` + 缩放 0.98
- 禁用：背景 `cream.200`，文字 `ink.400`
- 加载：左侧出 14px 转圈，文字保留

### 5.3 GhostButton（次要操作）
- 透明底，1.5px `peach.500` 描边，文字 `peach.500`

### 5.4 LogChip（快捷记录芯片）
- 高 44，圆角 pill，左图标右文案
- 默认：`cream.200` + `ink.900`
- 选中：`peach.300` + `peach.700`

### 5.5 InputField
- 高度 52，圆角 `radius.md`，背景 `cream.100`
- focus：描边 1.5px `peach.500` + `shadow.glow`
- 错误：描边 `rose.500`，下方 12sp 红字提示

### 5.6 BottomTabBar
- 4 个 Tab：今天 / AI 助手 / 数据 / 我的
- 选中态：图标 + 文字变 `peach.500`，下方加 4×4 圆点
- 不使用底部"加号 FAB"，因为快捷记录已在首页

### 5.7 Toast / Snackbar
- 位置：顶部下滑（避免挡到底部输入）
- 圆角 `radius.lg`，左侧色块 4px 表达类型

## 6. 图标 & 插画

- **图标**：1.5px 描边、圆头圆尾、统一 24px 视觉尺寸（线性圆头风格）
- **贴纸**：用于空状态 / 完成动效 / 引导（手绘水彩感的小插画：奶瓶、月亮、橘子、连衣裙）
- **避免**：emoji 表情堆叠、3D 拟物、写实摄影
- 推荐资源：Notion 风格的 [Lucide](https://lucide.dev) 改造圆头版本

---

## 7. 文案（Voice & Tone）

### 7.1 总体语气
- 称呼：默认"爸爸""妈妈"，不写"用户"。可在设置里改名。
- 第一人称："我帮你看了下…"——AI 助手口吻
- 不用感叹号轰炸，一个段落最多 1 个

### 7.2 文案对照

| 场景 | ❌ 不要 | ✅ 这样写 |
|---|---|---|
| 空状态 | 暂无数据 | 还没记，要不要记一笔今天的喂奶？ |
| 加载 | Loading… | 让我翻一翻笔记…… |
| 成功 | 操作成功 | 记好啦 ✓ |
| 报错 | 网络异常 | 网不太好，等会儿再试一下 |
| AI 不能给医学建议 | 我不能回答 | 这个我不太敢替医生说，建议你问下产检医生 |
| 提醒爸爸 | 任务待办 | 妈妈今天可能想吃酸的，下班顺路买点橘子？ |

### 7.3 数字与时间
- 时间用相对："2 小时前喂的"
- 数据要带"对比"："比昨天多睡 20 分钟"
- 周数加描述："孕 24 周——宝宝跟玉米差不多大"

## 8. 关键页面信息架构

### 8.1 首页（今天）
1. **顶部问候条**：早安，今天孕 24 周第 3 天 + 天气小贴纸
2. **今日重点卡（大卡片，cream.100）**：
   - 妈妈状态：胎动 / 体重 / 心情快记
   - 下一步：14:00 该提醒妈妈喝水了
3. **快捷记录区**：横向滚动 Chip（胎动 / 喝水 / 体重 / 产检 / 心情）
4. **AI 入口卡（peach.300 渐变）**：今天能吃 xxx 吗？→ 进入 AI

### 8.2 AI 助手（妈妈能吃什么）
- 顶部大输入框：「拍张照片或问我」
- 快捷气泡：螃蟹能吃吗 / 今天孕 24 周该补什么 / 这个药安全吗
- 回答卡：✅可吃 / ⚠️少量 / ❌避免，附理由 + 营养建议
- 永远带一句免责："这是参考，特殊情况问产检医生"

### 8.3 数据
- 顶部分段：体重曲线 / 胎动趋势 / 睡眠 / 摄入
- 一屏图表 + 一句话总结："最近 7 天平均胎动 32 次，正常"
- 图表：圆角柱 / 软曲线，颜色用 `sky.500` `mint.500`

### 8.4 提醒中心（爸爸专属）
- 时间轴样式：今天 / 这周 / 接下来
- 每条提醒带"快速操作"：已完成 / 推迟 1 小时 / 让妈妈知道
- 智能提醒来源：孕周节点（NT/糖耐）+ 妈妈记录的不舒服 + 季节性饮食

### 8.5 我的
- 宝宝/妈妈档案、技能开关、家庭成员、深色模式、隐私

## 9. 动效

- 所有过渡 200ms `ease-out-quart`
- 卡片进入：从下 8px 上滑 + 透明度 0→1
- 完成动作：贴纸图标弹一下（scale 0.8→1.05→1，250ms）
- 长列表懒加载：底部"鸡蛋仔在烤……"占位

---

## 10. 无障碍

- 文字最小 12sp，正文 ≥ 14sp
- 主 CTA 文字白底比 ≥ 4.5:1（已校验）
- 所有图标按钮必须有语义 `Semantics(label: …)`
- 支持系统字号放大到 200%，布局自适应
- 深色模式：底色 `#1F1A14`，cream 色相整体降亮 70%，强调色保持
