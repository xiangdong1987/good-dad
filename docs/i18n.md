# 多语言（i18n）

App 默认中文，目前预留了 `zh-CN / zh-TW / en / ja` 四种 locale，可在「设置 → 语言」实时切换。

## 架构

```
lib/core/i18n/
├── app_locale.dart        # 枚举 + 内部 code + label + 给 AI 的语言名
├── l10n.dart              # Map-based key→string，缺失自动回退到 zh-CN
└── locale_provider.dart   # AsyncNotifier 持久化到 secure_storage(app.locale)
```

`MaterialApp.router` 的 `locale` / `supportedLocales` / `localizationsDelegates` 由 `localeProvider` 驱动，切换语言时整树 rebuild。

## 在 UI 里使用

```dart
final l10n = ref.watch(l10nProvider);
Text(l10n.t('home.help_today'))
```

带参数：
```dart
l10n.t('home.greeting', {'name': '老周'})  // → "嘿，老周"
```

## 添加新 key

在 `lib/core/i18n/l10n.dart` 里：
1. 往 `_zhCN` 加 `'foo.bar': '中文文案'` —— 这是必需的（兜底字典）
2. 想翻译就在 `_en` / `_zhTW` / `_ja` 加同 key 的翻译
3. 不加翻译也能跑，会自动回退到中文

## 添加新语言

1. `app_locale.dart` 的 `AppLocale` 枚举加一项，写好 `code` / `label` / `aiLanguageHint`
2. `l10n.dart` 加对应 strings map
3. `_dict()` switch 加分支
4. 完事

## AI 跟着语言走

`SkillRunner` 在 system prompt 里追加：

> ## 语言
> 请用 English 回复用户。保留 JSON 字段名 / 枚举值（safe/avoid/caution/todo/checkup 等）不翻译。

所以即使 SKILL.md 全是中文写的，**AI 输出会按用户选的语言走**——结构化字段（verdict、kind 等枚举）保持英文以免破坏解析。

## 当前覆盖度

UI 文案目前**绝大多数仍是中文硬编码**，没有进 L10n。`l10n.dart` 里已经放了一批常用 key（设置、按钮、主页问候）作为种子。后续新增 UI 时优先用 `l10n.t(...)`，存量 UI 按需迁移。

## 不在范围里

- **SKILL.md 翻译**：当前 7 份 SKILL.md 全中文写。要做多语言版本可以在 `assets/skills/<name>/SKILL.<locale>.md`，让 SkillLoader 优先读 locale 对应文件，回退到默认。这是较大工程。
- **数字/日期格式本地化**：当前用硬编码格式。需要时再用 `intl` 的 `DateFormat`/`NumberFormat` 按 `locale` 格式化。
- **RTL**：阿拉伯语等 RTL 语言要测一遍 layout。当前未支持。
